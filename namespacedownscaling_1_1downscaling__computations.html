Applies downscaling to daily discharge data to generate sub-daily discharge flow durations.

This function takes daily discharge data and applies statistical downscaling techniques based on regression models, distribution fitting, and specified criteria to compute sub-daily discharge intervals.

Parameters
meteo_df(pandas.DataFrame) Meteorological data containing relevant predictors (e.g., temperature, precipitation) for criteria-based downscaling.
months(list of int) List of months to include in the analysis (e.g., [6, 7, 8, 9] for summer).
kde_dict(dict) Dictionary containing kernel density estimation (KDE) distributions for downscaling parameters (e.g., $a$, $b$, $c$, $M$).
function(str) Discharge model to use for curve fitting. Options: "Singh2014", "Sigmoid_d", "Sigmoid".
qmin_regr(sklearn model) Regression model for estimating the minimum daily discharge.
qmax_regr(sklearn model) Regression model for estimating the maximum daily discharge.
FDC_output_file(str) Path to the output CSV file where the generated flow duration curves (FDCs) will be saved.
file_paths(FilePaths) Object containing all file paths, used to give the file path and name for saving intermediate distribution files (e.g., $a$, $b$, $c$, $M$).
subdaily_nb(int) Number of sub-daily intervals per day (e.g., 96 for 15-minute intervals).
modeled(bool, optional) Whether the input discharge data is modeled or observed. Default is False.
criteria(list of str, optional) List of additional meteorological predictors to include in the downscaling process. Default is None.
debug(bool, optional) If True, enables additional assertions and debugging outputs. Default is False.
Returns
(pandas.DataFrame) A DataFrame containing the downscaled sub-daily discharge data with 15-minute intervals.

Notes:

  • The function supports multiple discharge fitting models and includes an option for weather-state-specific downscaling.
  • Missing discharge or meteorological data is handled by imputing NaNs into the generated distributions.
  • Distributions of downscaling parameters are saved as CSV files in the specified path for debugging or further analysis.
